# Rename `dotori` → `zemon` Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Rename the project from `dotori` to `zemon` across all code identifiers, manifests, environment variables, documentation, the GitHub repository, and the local working directory.

**Architecture:** Two code-level commits inside an isolated worktree — one for source/manifests/Cargo.lock (single buildable atomic), one for docs — followed by a grep gate to confirm zero stragglers, an `expect`-driven smoke test, merge to master, and finally a short list of post-merge external renames (GitHub repo rename, `git remote set-url`, local directory mv) the user performs.

**Tech Stack:** Rust workspace, Cargo, `git mv`, GNU `sed` (BSD `sed` on macOS — `-i ''` form), `grep`, `expect`.

**Spec:** `docs/superpowers/specs/2026-05-13-rename-zemon-design.md`

---

## File Structure

This plan modifies and renames files only — no new files.

| Path | Change | Responsibility |
|---|---|---|
| `crates/dotori-core/` → `crates/zemon-core/` | `git mv` | shared library |
| `crates/dotori-cli/` → `crates/zemon-cli/` | `git mv` | binary crate |
| `crates/dotori-tui/` → `crates/zemon-tui/` | `git mv` | TUI library |
| `Cargo.toml` (workspace) | edit | workspace members + `workspace.dependencies` |
| `crates/zemon-core/Cargo.toml` | edit | `[package].name` |
| `crates/zemon-cli/Cargo.toml` | edit | `[package].name`, `[[bin]].name`, `[dependencies]` |
| `crates/zemon-tui/Cargo.toml` | edit | `[package].name` |
| `crates/zemon-core/src/config.rs` | edit | `DotoriConfig` → `ZemonConfig`, `DOTORI_*` → `ZEMON_*` |
| All other `crates/**/*.rs` | edit | `dotori_core` → `zemon_core`, `dotori_tui` → `zemon_tui`, `DotoriConfig` → `ZemonConfig` |
| `Cargo.lock` | regenerated by `cargo build` | dependency lockfile |
| `README.md` | edit | project title, examples, env var names, crate layout |
| `CLAUDE.md` | edit | project description, build commands, paths |
| `docs/superpowers/specs/*.md` | edit body only | filenames preserved |
| `docs/superpowers/plans/*.md` | edit body only | filenames preserved |

## Task 1: Code rename — single buildable commit

**Files:** every Rust source and Cargo manifest in the workspace; produces a new `Cargo.lock`.

This task lands as **one commit**. Intermediate steps will not compile until the last sed pass completes — that is expected and the reason for the single-commit structure.

- [ ] **Step 1.1: Rename crate directories with `git mv`**

Run from worktree root:

```bash
git mv crates/dotori-core crates/zemon-core
git mv crates/dotori-cli crates/zemon-cli
git mv crates/dotori-tui crates/zemon-tui
```

Verify:

```bash
ls crates/
# Expected output (three lines):
# zemon-cli
# zemon-core
# zemon-tui
```

- [ ] **Step 1.2: Update workspace `Cargo.toml`**

Edit `Cargo.toml` (workspace root). Current content of the relevant sections:

```toml
[workspace]
resolver = "2"
members = [
    "crates/dotori-core",
    "crates/dotori-cli",
    "crates/dotori-tui",
]
...
[workspace.dependencies]
dotori-core = { path = "crates/dotori-core" }
dotori-tui = { path = "crates/dotori-tui" }
```

Change `members` to:

```toml
members = [
    "crates/zemon-core",
    "crates/zemon-cli",
    "crates/zemon-tui",
]
```

Change `workspace.dependencies` to:

```toml
dotori-core = { path = "crates/dotori-core" }   # delete this line
dotori-tui = { path = "crates/dotori-tui" }     # delete this line
```

Replace with:

```toml
zemon-core = { path = "crates/zemon-core" }
zemon-tui = { path = "crates/zemon-tui" }
```

Use `sed` to do this in one shot:

```bash
sed -i '' \
  -e 's|crates/dotori-core|crates/zemon-core|g' \
  -e 's|crates/dotori-cli|crates/zemon-cli|g' \
  -e 's|crates/dotori-tui|crates/zemon-tui|g' \
  -e 's|^dotori-core = |zemon-core = |' \
  -e 's|^dotori-tui = |zemon-tui = |' \
  Cargo.toml
```

Verify:

```bash
grep -n "dotori" Cargo.toml
# Expected: no output
```

- [ ] **Step 1.3: Update each crate's `Cargo.toml`**

Current `crates/zemon-core/Cargo.toml` has `name = "dotori-core"`. Same pattern for `cli` (which also has `[[bin]] name = "dotori"` and `[dependencies]` entries `dotori-core.workspace = true`, `dotori-tui.workspace = true`) and `tui`.

Run:

```bash

sed -i '' \
  -e 's|name = "dotori-cli"|name = "zemon-cli"|' \
  -e 's|name = "dotori"|name = "zemon"|' \

  -e 's|^dotori-tui\.workspace|zemon-tui.workspace|' \

  -e 's|name = "dotori-tui"|name = "zemon-tui"|' \
  -e 's|^dotori-core\.workspace|zemon-core.workspace|' \
  crates/zemon-tui/Cargo.toml
```

Verify:

```bash
grep -n "dotori" crates/*/Cargo.toml
# Expected: no output
```

- [ ] **Step 2.2: Apply replacements to docs**

Specific-before-generic ordering matters here too. The `dotori-zenoh` literal (project/repo name) becomes `zemon`, not `zemon-zenoh`, per spec.

```bash
find README.md CLAUDE.md docs/superpowers -name '*.md' -type f -exec sed -i '' \
  -e 's|gongfour/dotori-zenoh|gongfour/zemon|g' \
  -e 's|dotori-zenoh|zemon|g' \
  -e 's|DOTORI_ENDPOINT|ZEMON_ENDPOINT|g' \
  -e 's|DOTORI_MODE|ZEMON_MODE|g' \
  -e 's|DOTORI_NAMESPACE|ZEMON_NAMESPACE|g' \
  -e 's|DOTORI_CONFIG|ZEMON_CONFIG|g' \
  -e 's|DOTORI_SCOUT_PORT|ZEMON_SCOUT_PORT|g' \
  -e 's|DotoriConfig|ZemonConfig|g' \
  -e 's|dotori_core|zemon_core|g' \
  -e 's|dotori_tui|zemon_tui|g' \
  -e 's|dotori-core|zemon-core|g' \
  -e 's|dotori-cli|zemon-cli|g' \
  -e 's|dotori-tui|zemon-tui|g' \
  -e 's|`dotori`|`zemon`|g' \
  -e 's|dotori |zemon |g' \
  -e 's|^dotori$|zemon|g' \
  -e 's|/dotori |/zemon |g' \
  -e 's|/dotori$|/zemon|g' \
  -e 's|Dotori|Zemon|g' \
  {} +
```

The trailing replacements catch the bare `dotori` binary name in contexts like `./target/release/dotori`, command-line examples (`dotori sub ...`), and standalone occurrences. They are intentionally narrower than a blanket `dotori → zemon` to avoid mangling filenames inside the body text (the filenames `2026-04-14-dotori-cli-design.md` etc. should remain unchanged because the past-doc-filename decision is **A**, body-only).

Note: this leaves bare references to filenames like `2026-04-14-dotori-cli-design.md` inside the doc bodies untouched, which is correct — those are paths and they still exist on disk under the old filename.
