# Rename `dotori` → `zemon` — Design

**Date:** 2026-05-13
**Scope:** Rename the project from `dotori` to `zemon` across code, manifests, environment variables, documentation, GitHub repository, and the local working directory.

## Motivation

The name `dotori` is now used inside the author's company for a different product. To avoid ambiguity in conversations, code reviews, and search, the personal Zenoh-monitor project is renamed to `zemon` (a portmanteau of *zenoh* and *monitor*). Availability was verified before deciding:

- `crates.io`: `zemon`, `zemon-core`, `zemon-cli`, `zemon-tui` are all unclaimed.
- GitHub: `gongfour/zemon` does not exist.

## Goals

- Every identifier and document on the new project name reads consistently.
- The binary, crate names, import paths, type names, environment variables, and external (GitHub repo, local directory) names all align.
- The rename is atomic in the codebase — a single buildable commit, not an intermediate state.

## Non-goals

- Re-publishing or first-time publishing to crates.io.
- Adding new features or refactoring beyond the rename.
- Rewriting git history (commit messages keep their original wording).
- Migrating past spec/plan file names — only their body content is updated.

## Decisions (recorded from brainstorming)

| Question | Decision |
|---|---|
| New name | **`zemon`** (ze = zenoh, mon = monitor; also evokes *daemon*) |
| Scope | **A** — code, repo, local directory, all docs |
| Past spec/plan docs | **A** — body text updated, filenames preserved |
| Environment variables | **A** — hard switch `DOTORI_*` → `ZEMON_*`, no compatibility shim |

## Change matrix

| Category | Old | New | Notes |
|---|---|---|---|
| Crate names (kebab) | `dotori-core`, `dotori-cli`, `dotori-tui` | `zemon-core`, `zemon-cli`, `zemon-tui` | `crates/<name>/` directories also renamed via `git mv` |
| Rust paths (snake) | `dotori_core`, `dotori_tui` | `zemon_core`, `zemon_tui` | import paths inside source |
| Binary | `dotori` | `zemon` | `[[bin]] name` in `crates/zemon-cli/Cargo.toml` |
| Type | `DotoriConfig` | `ZemonConfig` | `crates/zemon-core/src/config.rs` and all callers |
| Env vars | `DOTORI_ENDPOINT`, `DOTORI_MODE`, `DOTORI_NAMESPACE`, `DOTORI_CONFIG`, `DOTORI_SCOUT_PORT` | `ZEMON_*` | hard switch; users update `.envrc`/scripts |
| Workspace manifest | members + workspace.dependencies | same change | `Cargo.lock` regenerated by `cargo build` |
| Current docs | `README.md`, `CLAUDE.md` | body text + paths/binary names updated | |
| Past docs | bodies of `docs/superpowers/specs/`, `docs/superpowers/plans/` | bodies updated, filenames preserved | file paths and `git log --follow` chain remain useful |
| GitHub repo | `gongfour/dotori-zenoh` | `gongfour/zemon` | `-zenoh` suffix dropped to match binary; GitHub creates a permanent redirect from the old URL |
| Local directory | `~/Projects/dotori-zenoh` | `~/Projects/zemon` | renamed by user outside the Claude session (or in a separate terminal) |
| Git remote | `git@github.com:gongfour/dotori-zenoh.git` | `git@github.com:gongfour/zemon.git` | `git remote set-url` after GitHub rename |

## Execution phases

### Phase 1: Code rename (branch work)

All changes land on a feature branch `feat/rename-zemon` in a separate worktree.

1. `git mv crates/dotori-{core,cli,tui} crates/zemon-{core,cli,tui}`
2. Update all 4 `Cargo.toml` files (workspace root + 3 crates): `name`, `path` deps, `[[bin]] name`.
3. Search-and-replace across all `.rs` files:
   - `dotori_core` → `zemon_core`
   - `dotori_tui` → `zemon_tui`
   - `dotori-core`, `dotori-cli`, `dotori-tui` (in comments, attributes) → `zemon-*`
   - `DotoriConfig` → `ZemonConfig`
4. Update `DOTORI_*` env vars → `ZEMON_*` in `crates/zemon-core/src/config.rs::from_env()` (5 variables).
5. Update body text in `README.md`, `CLAUDE.md`: project description, build commands, binary path examples, env var names, crate layout descriptions. The string `dotori-zenoh` (project/repo name) becomes `zemon`. The string `dotori` (binary) becomes `zemon`. References to "도토리" (if any) get the same treatment.
6. Update body text in `docs/superpowers/specs/*.md` and `docs/superpowers/plans/*.md`. Filenames are not changed. Replace every `dotori`-family identifier with its `zemon` equivalent. Leave commit-message excerpts in plans as-is when they quote actual commit messages from git history (the original commits keep their `dotori` wording).
7. Regenerate `Cargo.lock` via `cargo build`.
8. Verify:
   - `cargo build --release` — clean
   - `cargo test` — 30 tests pass
   - Smoke: `./target/release/zemon --mode peer pub test/x '{"y":1}'` succeeds (peer mode session opens without `zenohd`)
   - Smoke: `./target/release/zemon tui` launches without panic (auto-driven via `expect` like the previous branch's smoke test)
9. Single commit with all changes:
   ```
   chore: rename dotori → zemon

   New name (ze + mon for zenoh + monitor) replaces dotori, which now
   conflicts with an internal company project. crates.io and the target
   GitHub repo name were verified empty before choosing.
   ```

### Phase 2: Merge

Merge `feat/rename-zemon` into `master` using `--no-ff` to preserve a single, recognizable rename commit on the trunk.

### Phase 3: External renames (post-merge)

3.1 **GitHub repo rename** — performed by the user in the browser at `https://github.com/gongfour/dotori-zenoh/settings`. GitHub installs a permanent redirect from the old URL as long as `dotori-zenoh` is not recreated.

3.2 **Update git remote in local checkout**:
```bash
git remote set-url origin git@github.com:gongfour/zemon.git
git fetch  # confirm connectivity
```

3.3 **Local directory rename** — performed by the user outside the active Claude session:
```bash
cd ~/Projects
mv dotori-zenoh zemon
```

### Phase 4: Cleanup

- Remove the rename worktree via `git worktree remove`.
- Delete the merged feature branch.

## Risks and mitigations

| Risk | Mitigation |
|---|---|
| Intermediate non-buildable state during code rename | All changes in one commit; build/test gates after every change in the plan |
| Missed identifier (e.g. forgotten import or a rare occurrence in a doc) | After search-and-replace, run `grep -rn "dotori\|Dotori\|DOTORI" --include='*.rs' --include='*.toml' --include='*.md'` and require zero matches before committing |
| User scripts using old env vars silently fail | Documented in CLAUDE.md migration note; user is the only consumer, so impact is bounded |
| GitHub rename breaks tooling expecting the old URL | GitHub redirects persist; if any local clone still has the old URL it continues to work but should be updated |
| Crate name collision (e.g. someone publishes `zemon` to crates.io before we do) | We are not publishing; if/when publishing is decided, re-verify availability at that time |

## Tests

This is a rename, not a feature. No new tests are added. The success criterion is that the pre-existing 30 tests still pass under the new names.

## Out-of-scope follow-ups

- Logo, branding assets, screenshots.
- Updating any external references (blog posts, etc.) — none known.
- Setting up `cargo-release` or `cargo publish` workflow.
